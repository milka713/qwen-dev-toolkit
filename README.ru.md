# qwen-dev-toolkit

Набор практик для [qwen-code](https://github.com/QwenLM/qwen-code) на **локальных
моделях с маленьким контекстом** (например, llama.cpp с окном ~90–100k). Не даёт большим
сборкам ломаться после переполнения и лоссового компакта: делает модель
**архитектором, который делегирует**, а состояние — переживающим компакт.

> 🇺🇸 [English](README.md) · 🇷🇺 Русский

## Как работает

- **Делегирование** — главная сессия планирует и делегирует; вся тяжёлая работа (чтение
  файлов, написание кода, тесты) идёт в **одноразовых контекстах субагентов**, поэтому
  основной контекст мал и сборка доходит до конца. Субагентов сколько нужно, по одному на задачу.
- **Durable-состояние** — прогресс в `.qwen/PROGRESS.md`, хук переинжектит его каждую
  сессию (и после компакта), так что план не теряется.
- **Флаги, неуязвимые к компакту** — компакт сжимает только *переписку*; `QWEN.md`
  переподключается как системный контекст на каждый запрос. Поэтому блок в `QWEN.md`
  (например флаг режима разработки) всегда на месте и не требует переобъявления.

## Компоненты

### Команды (детерминированные, вызываются вручную)
Custom-команды — запись в файл идёт через shell-шаг (`!{…}`), поэтому детерминирована.

| Команда | Что делает |
| ------- | ---------- |
| `/dev` · `on` · `off` · `status` · `<цель>` | Переключатель режима разработки. `on` пинит блок в `QWEN.md` → сессия планирует и делегирует субагентам `implementer`; `off` убирает; `<цель>` включает **и** строит. Идемпотентно; остальной `QWEN.md` сохраняется. |
| `/cover` · `on` · `off` · `status` | Режим test-first / покрытия. Сборка **test-first** (red→green→refactor); покрытие **измеряется** (`pytest --cov`, `jest --coverage`, …) и должно быть **≥90% по изменённому коду** — проверенный результат, не «тыква». |
| `/pin <что угодно>` · `list` · `remove <текст>` · `clear` | Запомнить любую инфу (IP/порт, команду деплоя, решение, URL, сниппет) в непробиваемый компактом `FACTS.md` — автозагрузка через `@FACTS.md`, в `.gitignore` (не утечёт). Проектный. |
| `/status` | Read-only снимок: режимы dev/cover, число запиненных фактов, текущая цель и следующая задача. |

### Скиллы (вызываются и моделью, и вручную)

| Скилл | Что делает |
| ----- | ---------- |
| `/brainstorm` | Проясняет и прожимает требования **до** кода (объём, критерии готовности, edge-кейсы, ограничения), чтобы не потратить маленький контекст на «не то». Выдаёт согласованный спек и передаёт в `/plan`. |
| `/plan` | Превращает размытый/большой запрос в упорядоченный по зависимостям список задач в `.qwen/PROGRESS.md`. Разведка через read-only `scout`. Выдаёт план, не код. |
| `/implement` | Оркестратор: фиксирует цель, декомпозирует, гоняет **каждую задачу в свежем субагенте `implementer`**, затем проверяет end-to-end канонической командой. Для любой многошаговой сборки. |
| `/checkpoint` | Собирает важное состояние в `.qwen/PROGRESS.md`, чтобы пережить лоссовое авто-сжатие. `/checkpoint restore` перезагружает. |
| `/audit` | Аудит безопасности (архитектура + код): хардкод-секреты, authz, инъекции, SSRF, слабая крипта, рискованные зависимости — находки по серьёзности, чинит безопасные. |

### Субагенты (изолированный контекст)

| Субагент | Что делает |
| -------- | ---------- |
| `implementer` | Доводит **одну** задачу до проверенного состояния: читает реальные файлы, реализует полностью (без заглушек), проверяет **канонической** командой из корня (чинит упаковку, если проходит лишь через трюк с путём). Возвращает краткое резюме. |
| `scout` | Read-only исследователь — отдаёт компактный дайджест (ключевые файлы, связи, конвенции, реальные команды) вместо чтения файлов пачками в основной контекст. |

### Хуки (`~/.qwen/settings.json`)

| Хук | Что делает |
| --- | ---------- |
| `SessionStart` → `session-start-restore.js` | Переинжектит `.qwen/PROGRESS.md` в начале сессии / после компакта — модель восстанавливает цель и следующие шаги. |
| `PreCompact` → `pre-compact-steer.js` | Направляет встроенное сжатие сохранять цель, решения, список файлов и разбивку сделано/осталось. |
| `PreToolUse` → `secret-guard.js` | **Блокирует** любую запись/правку/команду с хардкод-секретом (приватные ключи, токены AWS/OpenAI/GitHub/Slack/HF, …) или коммит секрет-файла (`.env`, `id_rsa`, `*.pem`). Env-переменные и плейсхолдеры проходят. |
| `UserPromptSubmit` → `skill-reminder.js` | Маленькие локальные модели недо-вызывают скиллы; хук инжектит короткую точечную подсказку (напр. «похоже на безопасность → `/audit`) только при явном совпадении, чтобы нужный скилл реально срабатывал. На тривиальных промптах молчит. |

Плюс лаконичный `~/.qwen/QWEN.md` (режимы + дисциплина памяти) и нативная авто-память.

## Области видимости

`QWEN.md` — контекст-файл, переподключаемый на каждый запрос (не сжимается); грузится из
`~/.qwen/QWEN.md` (**глобально**) и `<проект>/QWEN.md` вверх по дереву (**проектно**).
Сессионного `QWEN.md` нет.

| Состояние | Где | Область |
| --------- | --- | ------- |
| Скиллы, субагенты, команды, хуки, гайд | `~/.qwen/…` | **Глобально** |
| Память `/pin` | `<проект>/FACTS.md` (в `.gitignore`) | **Проектно** |
| Флаги `/dev` и `/cover` | блок в `<проект>/QWEN.md` | **Проектно** (липко до `off`) |
| Состояние задач | `<проект>/.qwen/PROGRESS.md` | **Проектно** |

## Установка

```bash
./install.sh          # полная установка в ~/.qwen, затем перезапусти qwen-code
```

Копирует скиллы/субагентов/команды/хуки в `~/.qwen` и идемпотентно мёржит настройки хуков,
авто-памяти и гайд — не трогая твои ключи и настройки. Безопасно повторять.

Альтернативы: `qwen extensions install https://github.com/milka713/qwen-dev-toolkit`
(скиллы + субагенты + команды + гайд; для хуков запусти ещё `./install.sh`), либо скопируй
каталоги в `~/.qwen/` вручную.

**Проверка** (после рестарта): `/skills` показывает `implement, plan, checkpoint, audit`;
`/agents manage` — `implementer, scout`; `/status` отвечает.

## Использование

```text
/dev                                 # режим разработки (архитектор + делегирование)
/cover on                            # требовать реальные тесты, ≥90% покрытия
/pin модель-сервер 10.0.0.5:8080     # запомнить инфра-факты (компакт-пруф, в .gitignore)
/plan добавь JWT-авторизацию в API   # дизайн → .qwen/PROGRESS.md
/implement                           # сборка через субагентов
/audit                               # аудит безопасности
/checkpoint [restore]                # сохранить / перезагрузить состояние
/status                              # что включено, текущая цель
/dev off                             # обратно в режим одного агента
```

Одной строкой: `/dev собери Python CLI трекер расходов с SQLite и pytest`.

## Требования и удаление

- qwen-code (проверено на **0.19.x**) + Node.js. Любой провайдер; задуман для локальных моделей с маленьким контекстом.
- `./uninstall.sh` удаляет только файлы и блоки конфига этого набора; остальные настройки, env и `.qwen/PROGRESS.md` остаются.
